## 2. Classic Mode — Core Concept

The player builds a **Squad XI** of 11 fictional football players. Each squad member has a complete, procedurally generated career — debut club, stats progression, trophies won, and retirement.

### Session Structure

```
App
 └── Game Session (Game 1, Game 2, ...)
       └── Squad XI (11 slots in formation)
             └── Each slot → 1 Career Player (fully generated)
```

- Each **Game Session** is independent (like a new save file).
- A Squad XI has exactly **11 player slots**, arranged in a formation.
- Once all 11 slots are filled, the squad is complete.
- The user can: **Save**, **Delete**, or (future) **Use in another mode**.
- **Squad Rating** = Average of all 11 players' **Peak OVR**.

## 3. Player Generation — The Wheel Cascade

Each player is generated by spinning a series of wheels in sequence.
The output of each wheel feeds as input into subsequent wheels.

```
PHASE 1 — SETUP (Basic Identity Wheels)
──────────────────────────────────────────────────────────────
 ① Nationality Wheel
 ② Position Wheel           ← pre-determined by slot, cosmetic or skipped
 ③ Debut Age Wheel          ← bell curve centered around 17–18
 ④ Debut Stats Wheel        ← starting OVR (45–75), depends on age & position
 ⑤ Career Length Wheel      ← total years (8–28)
 ⑥ Number of Clubs Wheel    ← total clubs (1–8)

PHASE 2 — CAREER STINT & YEAR SIMULATION LOOP (Interactive Wheels & Auto calculations)
──────────────────────────────────────────────────────────────
 For each club stint (up to Number of Clubs):
   ⑦a League Wheel          ← select league, weighted by current OVR & nationality
   ⑦b Club Wheel            ← select club in that league, weighted by OVR vs club prestige
   ⑧  Years at Club Wheel   ← how many years player stays at this club (decrements remaining length)

   For each year in the stint:
     [Auto-Calculated by BE (No Spin)]
       - Match stats: Appearances (Apps), Goals (G), Assists (A) based on position & OVR
       - Average Match Rating (e.g. 6.2 - 8.6)
       - Team standings: League standings & cup competition results (domestic, continental)
     [Ballon d'Or Wheel]    ← optional, triggered if candidate is eligible (OVR > 85, high performance)
     [End of Year Stats Update Wheels]
       ⑨a Stats Change Wheel    ← OVR change direction: increase / decrease / maintain
       ⑨b Quantity of Stats Wheel← how many of the 6 core stats are affected (1–3 stats)
       ⑨c Stats Magnitude Wheel  ← value change for each affected stat (+1, +2, +3, -1, -2, etc.)
     [International NT Wheels (If international tournament year - every 2/4 years)]
       ⑩a Call-Up Wheel     ← see if player gets called to national team (based on OVR & form)
       ⑩b Tournament Wheel   ← NT performance (Group stage, QF, Winner, etc.)

PHASE 3 — AUTO GENERATION & RETROSPECTIVE COMPILATION
──────────────────────────────────────────────────────────────
 ⑪  Name                    ← generated from Nationality name pool
 ⑫  Height & Preferred Foot ← weighted by position
 ⑬  Hidden Stats            ← weighted random (personality, professionalism, luck)
 ⑭  Peak OVR Extraction     ← retrospectively fetched from the highest OVR season
                              (tie-broken by position-specific stats if duplicate OVRs)
```

## 4. Phase 1 — Identity Wheels

### 4.1 Nationality Wheel

- Pool: All nationalities represented in supported leagues (see Section 8).
- Weight: Equal distribution by default.
- **User Bias Input**: User can select 1–3 preferred nationalities before spinning. Each preferred nationality receives a +weight bonus (e.g., standard weight ×5).
- Output feeds: name generation, club tier affinity, initial league weight, and national team international tournament eligibility (e.g., Euro, Copa América, World Cup).

### 4.2 Debut Age Wheel

- **Range**: 15 – 22 years old.
- Weight distribution (bell curve, peak at 17–18):

| Age | Weight |
|---|---|
| 15 | 5% |
| 16 | 10% |
| 17 | 22% |
| 18 | 25% |
| 19 | 18% |
| 20 | 11% |
| 21 | 6% |
| 22 | 3% |

- Output feeds: debut OVR range, career length compatibility, retirement age.

### 4.3 Debut Stats Wheel

- **Debut OVR range** is determined by debut age + position:

| Debut Age | OVR Range |
|---|---|
| 15–16 | 45 – 58 |
| 17–18 | 55 – 66 |
| 19–20 | 60 – 72 |
| 21–22 | 65 – 75 |

- Position modifier applied to OVR range:
  - GK: +3 (GKs develop later)
  - CB/CDM: +2
  - ST/LW/RW: 0
  - CAM/CM: 0
  - LB/RB: +1

- **Position-based 6 Core Stats Distribution**:
  When generating the 6 core stats (PAC, SHO, PAS, DRI, DEF, PHY) during the debut setup, the wheel weights are dynamically modified based on the player's position to ensure realism:
  * **Strikers/Wingers (ST, LW, RW)**: Attack stats (SHO, PAC, DRI) are weighted higher (range 65–85). Defense stats (DEF) are forced to lower ranges (15–38).
  * **Defenders (CB, LB, RB)**: Defense and Physical stats (DEF, PHY) are weighted higher (65–85). Attack stats (SHO) are forced to lower ranges (15–40).
  * **Goalkeepers (GK)**: Defense (GK reflexes/positioning) and Physical (PHY) are weighted higher (65–88). Attack and Dribbling stats (SHO, DRI) are forced to lower ranges (10–30).
  * **Midfielders (CDM, CM, CAM)**: Passing and Dribbling stats (PAS, DRI) are weighted higher (65–85). CDM has a higher weight for DEF, while CAM has a higher weight for SHO.

- **Weighted OVR Calculation**:
  The Overall Rating (OVR) of the player is not calculated as a simple average, but as a weighted average prioritizing the key stats of their position:
  * **GK**: OVR = DEF * 0.60 + PHY * 0.30 + PAS * 0.10
  * **CB**: OVR = DEF * 0.45 + PHY * 0.35 + PAC * 0.15 + PAS * 0.05
  * **LB / RB**: OVR = PAC * 0.30 + DEF * 0.30 + PAS * 0.20 + DRI * 0.10 + PHY * 0.10
  * **CDM**: OVR = DEF * 0.35 + PHY * 0.30 + PAS * 0.20 + DRI * 0.10 + PAC * 0.05
  * **CM**: OVR = PAS * 0.30 + DRI * 0.25 + DEF * 0.15 + PHY * 0.15 + SHO * 0.10 + PAC * 0.05
  * **CAM**: OVR = PAS * 0.35 + DRI * 0.30 + SHO * 0.25 + PAC * 0.10
  * **LW / RW**: OVR = PAC * 0.35 + DRI * 0.30 + SHO * 0.20 + PAS * 0.15
  * **ST**: OVR = SHO * 0.45 + PAC * 0.25 + DRI * 0.15 + PHY * 0.10 + PAS * 0.05

- Output feeds: starting stats snapshot for the year-by-year timeline.

### 4.4 Career Length Wheel

- **Range**: 8 – 28 years.
- Must be compatible with debut age (debut_age + career_length ≤ 43 max retire age).
- Weight distribution (bell curve, peak at 14–18 years):

| Length | Weight |
|---|---|
| 8 – 10 yrs | 10% |
| 11 – 13 yrs | 20% |
| 14 – 18 yrs | 40% |
| 19 – 22 yrs | 20% |
| 23 – 28 yrs | 10% |

- Output feeds: club stint distribution, retirement age display.

### 4.5 Number of Clubs Wheel

- **Range**: 1 – 8 clubs.
- Weight distribution:

| # Clubs | Weight | Archetype |
|---|---|---|
| 1 | 5% | One-club legend |
| 2 – 3 | 35% | Focused career |
| 4 – 5 | 40% | Typical journeyman |
| 6 – 8 | 20% | True journeyman |

- Constraint: number_of_clubs ≤ career_length (can't have more clubs than years).
- Output feeds: club stint year allocation.

## 5. Phase 2 — Career Simulation Loop (Stint & Year Loop)

The player's career is simulated iteratively. The total career duration is tracked. Each club stint consumes a portion of the remaining career years, until the career length is exhausted.

### 5.1 Stint Level Wheels (League, Club, & Years)

For each of the $N$ clubs (where $N$ = Number of Clubs):
1. **League Wheel**: The player spins to determine which league to join. The wheel is populated with leagues whose tiers align with the player's *current OVR* at that age (see table below). Players get a +15% weight for their home country's leagues.

| OVR Range | Eligible League Tiers | Examples |
|---|---|---|
| 45 – 59 | Tier 3 / Developing leagues | J2 League, Liga MX lower, Asian leagues |
| 60 – 68 | Tier 2 / Mid leagues | Championship, Ligue 2, Brasileirão |
| 69 – 75 | Tier 1 / Strong leagues | Eredivisie, Süper Lig, Saudi Pro League, J1 |
| 76 – 82 | Tier 1 / Top leagues | Serie A, Bundesliga, Ligue 1, Liga Portugal |
| 83 – 87 | Tier 1 / Big 5 top half | Premier League, La Liga, Serie A (top clubs) |
| 88+ | Tier 1 / Elite clubs only | Real Madrid, Man City, Bayern, PSG tier |

2. **Club Wheel**: The player spins to select a specific club from the chosen league's club pool, weighted by club prestige and OVR compatibility.
3. **Years at Club Wheel**: The player spins to determine how many years they will stay at this club.
   - *Constraint*: Minimum 1 year. Must not exceed the remaining career years minus $(N - \text{current\_stint})$.
   - For the final club stint ($N$), the years are automatically set to the remaining career years (no spin).

### 5.2 Year-by-Year Simulation Loop

Within each stint, for each year, the game processes the following sub-steps:

#### A. Interactive Team & Cup Wheels (Spins)
The team achievements are determined by weighted wheels spun by the player:
1. **League Standing Wheel**: Determines the club's league position (1 to 20).
   - *Weights*: Controlled by club prestige and player OVR (higher player OVR compared to league average increases chance of top spots - "carrying" the team).
2. **Domestic Cup Wheel**: Determines the club's cup progress (Winner, Runner-Up, Semi-Finals, Early Exit).
   - *Weights*: Driven by club prestige, player OVR, and player `luckRating`.
3. **Continental Cup Wheel**: Only spun if the club qualified for a continental tournament in the current season (decided by the league standing achieved in the *previous* season, see Section 7.3 for slot rules).
   - *Trophies list*: Displays the actual cup name according to the club's geography (UEFA Champions League/Europa League/Conference League, Copa Libertadores, AFC Champions League, CONCACAF Champions Cup).
   - *Outcomes*: Winner, Runner-Up, Semi-Finals, Group Stage.

#### B. Personal Match Performance (BE Auto-Calculation, No Spin)
The BE automatically computes the player's personal seasonal stats based on their position, current OVR, and club prestige:
- **Match Stats**: Total Appearances (Apps), Goals (G), and Assists (A).
- **Match Rating**: Average rating of the season (e.g., 6.2 to 8.8) and clean sheets for GKs/defenders.
- **Individual Milestones**: Golden Boot, Golden Glove, Team of the Season.

#### C. Ballon d'Or Wheel (Optional)
If the player's OVR is $>85$ and they have an outstanding season (high Goals/Assists/Clean Sheets and Match Rating $\ge 7.80$, plus continental/international titles), they are nominated for the Ballon d'Or.
- **Ballon d'Or Wheel**: User spins to see if they win the Ballon d'Or. Winning weight is controlled by OVR, performance rating, and `luckRating`.

#### D. End-of-Year Stats Update Wheels
To simulate natural progression and decline, three wheels are spun at the end of each year:
1. **OVR Change Direction Wheel**: Decide if OVR increases, decreases, or remains unchanged (increase / decrease / maintain).
   - *Formula*: `Final Weight = Base Weight + Age Modifier + Performance Modifier`.
   * **Age Modifier**: Young players (17-22) have high increase weights; Prime players (23-28) maintain; Veterans (30+) decline fast (especially PAC/PHY).
   * **Performance Modifier**: High Match Rating ($\ge 7.50$) increases chance of stats improvement; low rating increases chance of decline. Trophies add a luck bonus.
2. **Quantity of Stats Wheel**: If OVR change is not 'maintain', this wheel determines how many of the 6 core stats are affected (1 to 4 stats).
3. **Stats Magnitude Wheel**: Spins to determine value change (+/- 1 to 4 points). Affected stats are dynamically prioritized based on player position (e.g. CB prioritizes DEF/PHY, ST prioritizes SHO/PAC/DRI).

#### E. International National Team Wheels (Every 2/4 Years)
During World Cup or continental cup years:
1. **Call-Up Wheel**: Determines if the player is selected for the national team squad. Weight depends on player OVR, season form/Match Rating, and the nation's Tier.
2. **Tournament Performance Wheel**: If called up, spins to simulate the national team's run (Group Stage, Round of 16, QF, SF, Runner-Up, Winner). Positive outcomes add national team trophies and tournament awards (e.g. Euro Golden Boot) to the player's career event log.

## 6. Phase 3 — Auto Generation (Computed, No Spin)

### 6.1 Name Generation

- Pulled from a **nationality-specific name pool** (first name + surname).
- Name pools exist for every supported nationality.
- Example pools:
  - Brazil: Gabriel, Lucas, Matheus / Silva, Santos, Ferreira
  - England: Jack, Harry, Mason / Smith, Jones, Wilson
  - Japan: Keito, Ryo, Takumi / Yamamoto, Tanaka, Sato

### 6.2 Physical Attributes

| Attribute | Generation Logic |
|---|---|
| Height | Weighted by position: GK avg 190cm, CB avg 187cm, ST avg 182cm, LW/RW avg 175cm |
| Preferred Foot | Right: 75%, Left: 20%, Both: 5% (position-independent) |

### 6.3 Year-by-Year OVR Progression & Peak OVR

Instead of a pre-calculated mathematical curve, the player's OVR progression is **dynamic and interactive**, driven by the End-of-Year Stats Update wheels spun at the end of each season.

- **OVR Timeline**: The player's OVR history is recorded season-by-season (e.g. Age 17: 62 OVR, Age 18: 65 OVR, Age 19: 68 OVR, etc.).
- **Peak OVR Extraction**: Once the career length is fully exhausted (player retires), the system retrospectively inspects the OVR timeline.
  - **Primary criterion**: The season with the highest overall OVR is designated as the `peakOvr`.
  - **Tie-breaker**: If multiple seasons share the same highest OVR, the system looks at the position-specific core stats (e.g., Shooting/Pace for Strikers, Defending/Physical for Centre Backs) to determine which season represents the player's absolute peak.
  - **Card Generation**: The player's FIFA-style card is rendered using the attributes from this peak season.

### 6.4 Sub-Stats (FIFA-Style 6 Attributes)

Generated from OVR at each age, weighted by position archetype:

| Position | PAC | SHO | PAS | DRI | DEF | PHY |
|---|---|---|---|---|---|---|
| GK | — | — | — | — | — | — |
| CB | Low | Low | Med | Low | **High** | High |
| LB/RB | Med | Low | Med | Med | High | Med |
| CDM | Med | Low | Med | Med | **High** | High |
| CM | Med | Med | **High** | Med | Med | Med |
| CAM | Med | High | High | **High** | Low | Med |
| LW/RW | **High** | High | Med | **High** | Low | Med |
| ST | High | **High** | Med | High | Low | Med |

> GK has its own stat set: Diving, Handling, Kicking, Reflexes, Speed, Positioning.

### 6.5 Hidden Stats

Generated once at player creation. Never shown to user. Influence future career events.

| Stat | Range | Effect |
|---|---|---|
| `personality` | 0 – 100 | Media events, dressing room events, transfer behavior |
| `professionalism` | 0 – 100 | Injury recovery speed, form consistency, longevity bonus |
| `luckRating` | 0 – 100 | Weight modifier on all positive/negative event outcomes |

## 7. Career Events — Club Stint Events

During each club stint, **career events are generated** based on:

1. **Real-world trophy history** of the club during the years the player was there.
2. **Player's OVR** at that time (determines role: key player, rotation, bench).
3. **Hidden Stats** (luck, professionalism influence outcomes).

### 7.1 Event Categories

| Category | Examples |
|---|---|
| **Trophy** | League Champion, Cup Winner, Continental Winner, Runner-Up |
| **Individual Award** | Top Scorer, Player of the Season, Golden Boot |
| **Performance** | Key Player, Rotation Player, Bench Warmer |
| **Career Milestone** | 100 Apps, 50 Goals, Career Goal, Debut Goal |
| **Transfer** | Loan, Permanent Transfer, Free Transfer, World Record |
| **Incident** | Injury (minor/major), Suspension, Form Dip |

### 7.2 Trophy Linkage to Real Club Data

Each club in the database has:

```ts
type ClubTrophyHistory = {
  leagueTitles: number;          // total domestic league titles
  domesticCups: number;          // FA Cup, Copa del Rey, DFB-Pokal, etc.
  continentalTitles: number;     // UCL, Copa Libertadores, AFC CL, etc.
  continentalType: 'UCL' | 'UEL' | 'UECL' | 'Libertadores' | 'AFC_CL' | 'CAF_CL' | 'CONCACAF_CL' | 'none';
  prestige: 1 | 2 | 3 | 4 | 5;  // 5 = elite (Real Madrid), 1 = lower league
}
```

Probability of trophy event per season at a club = `f(club.prestige, player.ovr, player.luckRating)`

### 7.3 National Team & International Tournament Events

The player's **Nationality** determines which international competitions they can participate in:

| Confederation | Nationalities (Examples) | Continental Tournament | Global Tournament |
|---|---|---|---|
| **UEFA** | England, Spain, Germany, Italy, France, Portugal, Netherlands, Belgium, Scotland | **UEFA Euro** | **FIFA World Cup** |
| **CONMEBOL** | Brazil, Argentina, Colombia, Ecuador, Uruguay, Chile | **Copa América** | **FIFA World Cup** |
| **AFC** | Japan, South Korea, Saudi Arabia, UAE, China, Australia | **AFC Asian Cup** | **FIFA World Cup** |
| **CONCACAF** | USA, Mexico | **CONCACAF Gold Cup** | **FIFA World Cup** |
| **CAF** | Egypt, South Africa | **CAF Africa Cup of Nations** | **FIFA World Cup** |

#### Call-Up & Simulation Logic
International tournaments are simulated on a **2-year/4-year cycle** mapped directly to the player's age to represent the real world tournament cycle:
- **FIFA World Cup**: Occurs at ages divisible by 4 (20, 24, 28, 32, 36...).
- **Continental Cups**: Occurs at even ages not divisible by 4 (18, 22, 26, 30, 34...).

1. **National Team Call-Up Threshold**:
   - A player is eligible for national team selection only if their **current OVR** at the time of the tournament meets the country's depth threshold:
     - *Tier 1 Nations* (e.g., England, France, Spain, Germany, Italy, Brazil, Argentina): OVR $\ge 80$.
     - *Tier 2 Nations* (e.g., Portugal, Netherlands, Croatia, Morocco, Senegal, Ivory Coast, USA, Mexico, Colombia, Uruguay, Japan, South Korea): OVR $\ge 75$.
     - *Tier 3 Nations* (e.g., Scotland, Saudi Arabia, South Africa, Turkey, Norway, Ghana, Canada, Cape Verde): OVR $\ge 70$.
2. **Simulation Outcomes**:
   - Eligible players spin the **Call-Up Wheel** to see if they get called to the national team squad (weights depend on player OVR and Match Rating of the previous season).
   - If selected, they spin the **International Tournament Wheel** (Group Stage, Quarter-Finals, Semi-Finals, Runner-Up, Winner). Outcomes depend on nation tier, player OVR, and `luckRating`.
   - Winners receive trophy events (e.g., *"FIFA World Cup Winner"*, *"UEFA Euro Winner"*). Plus, star players can win individual awards (e.g. *"World Cup Golden Boot"*).

### 7.4 Club Continental Qualification Logic

The continental tournament that the player's club participates in for the next season is determined dynamically by the club's geography and the **League Standing** achieved in the previous season, weighted by **League Prestige** (prestige rating from 1 to 5 stars in the database):

#### A. Europe (UEFA Leagues)
*   **League Prestige = 5** (e.g. Premier League, La Liga, Serie A, Bundesliga):
    *   Hạng 1 - 4: UEFA Champions League (C1)
    *   Hạng 5 - 6: UEFA Europa League (C2)
    *   Hạng 7: UEFA Conference League (C3)
*   **League Prestige = 4** (e.g. Ligue 1, Eredivisie, Primeira Liga):
    *   Hạng 1 - 2: UEFA Champions League (C1)
    *   Hạng 3 - 4: UEFA Europa League (C2)
    *   Hạng 5: UEFA Conference League (C3)
*   **League Prestige = 3** (e.g. Scottish Premiership, Süper Lig, Jupiler Pro League):
    *   Hạng 1: UEFA Champions League (C1)
    *   Hạng 2: UEFA Europa League (C2)
    *   Hạng 3: UEFA Conference League (C3)
*   **League Prestige $\le 2$** (Second-tier leagues):
    *   No continental qualification.

#### B. South America (CONMEBOL Leagues)
*   **League Prestige = 5** (Série A Brazil): Hạng 1 - 6 qualify for Copa Libertadores.
*   **League Prestige = 4** (Primera División Argentina): Hạng 1 - 4 qualify for Copa Libertadores.
*   **League Prestige $\le 3$**: Hạng 1 - 2 qualify for Copa Libertadores.

#### C. Asia (AFC) & North America (CONCACAF Leagues)
*   **League Prestige $\ge 4$** (Saudi Pro League, J1 League, MLS, Liga MX): Hạng 1 - 3 qualify for AFC Champions League / CONCACAF Champions Cup.
*   **League Prestige $\le 3$**: Only Hạng 1 (Champion) qualifies.

## 10. Wheel Spin UX

**Experience: Step-by-step — each wheel spins individually in sequence.**

### Flow per player slot

```
User clicks empty slot [ST]
        │
        ▼
┌─────────────────────────────────────┐
│  WHEEL 1: NATIONALITY               │
│  🌍 Wheel animates and spins...     │
│  → Result: 🇧🇷 Brazil               │
│  [Continue →]                       │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│  WHEEL 2: DEBUT AGE                 │
│  📅 Wheel animates and spins...     │
│  → Result: 17 years old             │
│  [Continue →]                       │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│  WHEEL 3: DEBUT STATS               │
│  📊 Wheel animates and spins...     │
│  → Result: OVR 62                   │
│  [Continue →]                       │
└─────────────────────────────────────┘
        │
       ... (Career Length, Clubs, per-club wheels)
        │
        ▼
┌─────────────────────────────────────┐

        │
        ▼
  Player card appears in slot with reveal animation
```

### Wheel UI Component Rules

- Each wheel is displayed as a **spinning visual wheel** with segments.
- After the spin resolves, the **winning segment highlights** and a result label appears.
- User must press **Continue** (or a key) to proceed to the next wheel.
- A **progress indicator** shows which wheel step they are on (e.g., Step 3 of 7).
- The previously resolved wheels are shown as a **summary strip** above the active wheel.
- User can **NOT go back** to a previous wheel once confirmed.

## 12. Data Model Summary

```ts
// Core entities for Classic Mode

type GameSession = {
  id: string;
  name: string;           // "Game 1", "Game 2", or user-defined
  createdAt: Date;
  squad: (CareerPlayer | null)[];  // length 11, null = empty slot
  squadRating: number | null;      // null until all 11 filled
  formation: Formation;
}

type CareerPlayer = {
  id: string;
  // Identity
  name: string;
  nationality: string;
  position: Position;
  height: number;
  preferredFoot: 'Right' | 'Left' | 'Both';
  // Career arc
  debutAge: number;
  retireAge: number;
  careerLengthYears: number;
  // Stats
  debutOvr: number;
  peakOvr: number;
  statsTimeline: StatsSnapshot[];   // Stats & OVR at each age
  // Club history
  clubStints: ClubStint[];
  // Hidden
  hiddenStats: HiddenStats;
  // Card meta
  cardRarity: CardRarity;
}

type ClubStint = {
  clubId: string;
  clubName: string;
  leagueId: string;
  leagueName: string;
  startAge: number;
  endAge: number;
  yearsAtClub: number;
  ovrAtJoining: number;
  ovrAtLeaving: number;
  events: CareerEvent[];
}

type StatsSnapshot = {
  age: number;
  ovr: number;
  pac: number;
  sho: number;
  pas: number;
  dri: number;
  def: number;
  phy: number;
  // GK stats (optional, populated only for GKs)
  div?: number;
  han?: number;
  kic?: number;
  ref?: number;
  spd?: number;
  pos?: number;
}

type CareerEvent = {
  type: 'trophy' | 'individual_award' | 'milestone' | 'incident' | 'transfer';
  label: string;           // "La Liga Champion", "Top Scorer", "100 Apps"
  age: number;
  clubId?: string;         // Optional, null/undefined for international national team events
  nationality?: string;    // Optional, populated for international national team events
}

type HiddenStats = {
  personality: number;       // 0–100
  professionalism: number;   // 0–100
  luckRating: number;        // 0–100
}

type CardRarity = 'bronze' | 'silver' | 'gold' | 'rare_gold' | 'epic' | 'legendary';
```
